Time-Cost Trade Off

13 Solved Problems 2069–2081

TCTO Technique Summary

Standard Procedure for Time-Cost Trade Off

Step 1: Draw the Network & Find Critical Path

Using predecessors/successors, draw the activity-on-node (AON) network. Perform forward pass (ES, EF) and backward pass (LS, LF) to find the critical path (activities with zero total float).

Step 2: Calculate Cost Slope for Each Activity

$$\text{Cost Slope} = \frac{\text{Crash Cost} - \text{Normal Cost}}{\text{Normal Duration} - \text{Crash Duration}}$$

This gives the additional direct cost per unit time saved by crashing.

Step 3: Identify Crashing Candidates

Only activities on the critical path can reduce project duration. Select the critical activity with the lowest cost slope.

Step 4: Crash & Check Feasibility

Crash by 1 unit at a time. After each crash: recalculate project duration, check for new critical paths. If multiple critical paths exist, crash activities on all critical paths simultaneously.

Step 5: Calculate Total Cost at Each Stage

$$\text{Total Cost} = \text{Total Direct Cost} + \text{Total Indirect Cost}$$

$$\text{Total Indirect Cost} = \text{Indirect Cost/unit} \times \text{Project Duration}$$

Continue crashing as long as cost slope < indirect cost per unit time.

Step 6: Identify Optimum Duration

Optimum Duration: Duration at which Total Cost is minimum.
Minimum Duration: Duration when all critical activities are fully crashed.
