Chapter 6

Vector Analysis

1. Data Management Functions vs. Data Analysis Functions Q1

  • Data Management Functions: These are operations used to organize, maintain, refine, and manipulate the spatial database without necessarily creating new spatial geometries or solving analytical problems. They prepare the dataset for analysis. Examples include data import/export, data editing (adding/modifying vertices), spatial data validation (topology checking), map projection/coordinate transformation, and attribute editing.
  • Data Analysis Functions: These functions use the geometric properties of vector data (points, lines, polygons) to derive insights, solve spatial problems, or create entirely new geographic datasets based on queries and mathematical operations. Examples include buffer analysis, network analysis, topological overlay (Clip, Intersect, Union), and point-in-polygon analysis.

2. Three Vector Data Management Functions Q2

If asked to explain strict data management operations, use these three:

  1. Data Conversion: Transforming spatial data between different models or formats, such as converting a vector shapefile into a raster grid (Rasterization) or converting CAD files into GIS shapefiles.
  2. Data Editing: Modifying existing vector features to correct errors or update outdated spatial and attribute data. This includes moving vertices, reshaping polygons, or updating the text fields in the attribute table.
  3. Topology Building/Validation: Establishing and checking the spatial relationships between adjacent or connected features to ensure data integrity. This involves applying rules like "polygons must not have gaps" or "lines must connect at nodes" to clean up digitizing errors.

3. Detailed Geoprocessing and Vector Functions Q3, Q4, Q5, Q6, Q7, Q8, Q13

The examiners frequently ask you to explain these specific functions along with their spatial maps and attribute table effects.

A. Clip

  • Concept: Acts like a "cookie cutter." It extracts features from an input layer that fall strictly within the spatial boundary of a clip layer.
  • Spatial Map: If the Input is a large irregular polygon (e.g., a forest) and the Clip layer is a perfect circle (e.g., a study area), the Output is only the piece of the forest that fits inside the circle.
  • Attribute Table: The output layer retains only the attributes of the Input layer. The Clip layer's attributes are ignored completely.

B. Intersect

  • Concept: Computes the geometric intersection of two layers. Only those features that overlap in both layers are preserved in the output.
  • Spatial Map: If Polygon A (Land Use) overlaps Polygon B (Flood Zone), the Output is strictly the overlapping middle section.
  • Attribute Table: The output attribute table joins the attribute columns from both Polygon A and Polygon B for the overlapping region.

C. Union (A Spatial Analysis / Topological Overlay Tool)

  • What it does: It takes two overlapping polygon layers and physically cuts them against each other. Every place where boundaries cross, a completely new, smaller polygon is created.
  • The Attribute Table: It joins the columns of Table A and Table B together (horizontally). Every new polygon knows what it was in Layer A and what it was in Layer B.
  • Rule: Both input layers must be polygons.
  • Civil Engineering Example: You have a polygon layer of "Soil Types" and another polygon layer of "Land Use" for the exact same watershed. You use Union to find every unique combination of soil and land use. If an "Agricultural" polygon overlaps a "Clay Soil" polygon, the Union operation cuts out that specific overlapping shape and its attribute table will read: Land Use: Agricultural | Soil: Clay.

D. Dissolve

  • Concept: Aggregates adjacent features or polygons that share the same attribute value into a single, larger feature.
  • Spatial Map: If you have 10 adjacent small square parcels, and 8 of them share the attribute "Residential", the output merges those 8 squares into one large "Residential" polygon boundary.
  • Attribute Table: The records are aggregated. Multiple rows are combined into a single record, reducing the total number of rows in the table.

E. Buffer

  • Concept: Creates a proximity zone (a new polygon) around a map feature (point, line, or polygon) measured at a specific uniform or variable distance.
  • Spatial Map: A line representing a highway will output a thick polygon tracing that highway at a 50m distance on both sides.
  • Attribute Table: A new polygon feature is created. The attribute table typically inherits the base feature's attributes and adds a new column stating the buffer distance (e.g., "BUFF_DIST = 50").

F. Merge (A Data Management Tool)

  • What it does: It takes two completely separate datasets (usually adjacent to each other) and groups them into one single file. It does not calculate intersections or break polygons into smaller pieces.
  • The Attribute Table: It simply takes the rows from Table B and adds them to the bottom of Table A (stacking them vertically).
  • Rule: The input layers must be the same geometry type (e.g., you can only Merge lines with lines, or polygons with polygons).
  • Civil Engineering Example: You have a shapefile of the "Kathmandu Road Network" and a separate shapefile of the "Lalitpur Road Network." You use Merge to create a single shapefile called "Valley Road Network."

G. Erase

  • Concept: Removes portions of the input feature that overlap with the "erase" feature. It is the geometric opposite of a Clip.
  • Spatial Map: If the Input is a large polygon and the Erase layer is a smaller overlapping circle, the Output is the large polygon with a blank "hole" punched out where the circle was.
  • Attribute Table: Retains only the input layer's attributes; the records and geometry for the erased portions are permanently removed.

2. Map Join / Spatial Join (A Data Analysis Tool)

  • What it does: It transfers attributes from one layer to another based purely on their spatial relationship (location) (like "intersects," "is completely within," or "is closest to"). It does not alter the geometric shapes of the target layer at all.
  • The Attribute Table: It joins the columns of Table B to the side of Table A (widening it horizontally) based on physical location.
  • Rule: The input layers can be different geometry types (e.g., you can join Polygon attributes to Point features).
  • Engineering Example: For your final year project on Site Response Analysis, you have a point shapefile containing 50 individual soil borehole locations from Kathmandu Geo Lab. You also have a polygon shapefile of the Kathmandu municipal wards. If you use a Spatial Join, the software looks at where each borehole point falls physically, and automatically adds the "Ward Number" and "Ward Population" columns from the polygon map directly into your borehole point attribute table.

I. Update

  • Concept: Computes the geometric intersection and replaces the input layer's features and attributes with the update layer's features and attributes where they overlap.
  • Spatial Map: A base map of land use, and a smaller polygon of a "New Airport Development". The Output shows the base map, but the area under the new development is entirely replaced by the new polygon's shape.
  • Attribute Table: The attributes of the base layer are overwritten by the attributes of the update layer purely in the overlapping region.

J. Split

  • Concept: Breaks a single input layer into multiple separate output files using the boundaries of a split layer.
  • Spatial Map: A map of rivers crossing through three different district polygons. The Output results in three completely separate files (e.g., River_DistA.shp, River_DistB.shp, River_DistC.shp).
  • Attribute Table: The original attribute table is divided into multiple smaller tables, one for each new output file.

4. Differentiating Specific Functions Q9

  • Clip vs. Intersect: Both isolate the overlapping spatial area. However, Clip only retains the attribute table of the input layer (acting merely as a cookie-cutter), whereas Intersect joins the attribute tables of both layers together in the output.
  • Merge vs. Dissolve: Merge combines two or more separate datasets into a single dataset without altering the individual feature geometries or combining rows (it just stacks the tables). Dissolve works within a single dataset, removing internal boundaries between adjacent polygons that share the same attribute value, thereby combining multiple rows into one.

5. Prove that Intersection = Union + Clip Q6e

This mathematical/logical proof requires you to break down the operations step-by-step:

  1. Let Layer A be the Input Layer and Layer B be the secondary layer.
  2. The Union Step: Performing a Union on Layer A and Layer B results in a combined map containing three distinct geometric areas: A unique, B unique, and the Overlap (A ∩ B). Crucially, the attribute table now contains the joined columns of both A and B.
  3. The Clip Step: Take the output of the Union and Clip it using Layer B's exact boundary as the cookie-cutter.
  4. Clipping to Layer B removes anything outside of B (deleting A unique). We are left with only the area inside B. Since the problem asks for the interaction of A and B, clipping the unioned layer strictly to the overlapping zone isolates the spatial geometry of (A ∩ B).
  5. Conclusion: The final geometry is just the overlapping area. Because the first step was a Union, the attribute table contains the joined data from both A and B. A geometric overlap that contains the joined attributes of both input layers is the exact definition of an Intersection. Therefore, Union + Clip = Intersection.

Exam Examples for Each Geoprocessing Function

A. Clip

Exam Example: You are given a massive geographic dataset of the entire national river network, but your final year project only focuses on a specific watershed model. You use the Clip tool, with your specific watershed boundary as the "cookie-cutter," to extract only the streams and rivers that flow within your study area. The resulting layer contains only the relevant river segments, keeping the original river attributes intact.

B. Intersect

Exam Example: You are conducting a site response analysis to identify areas highly vulnerable to earthquake damage. You have one polygon layer showing "High Soil Liquefaction Potential" and another polygon layer showing "Dense Urban Settlements." By applying the Intersect tool, the output map will strictly show the overlapping zones—representing highly populated areas built on dangerous soil. The output attribute table merges the soil properties and the population data for those specific areas.

C. Union

Exam Example: You are calculating the Runoff Curve Number for a hydrologic model (like mHM or HEC-HMS). You need to know the unique combinations of soil type and land use. You perform a Union on a "Soil Hydrologic Group" layer and a "Land Cover" layer. The output retains all spatial boundaries from both maps. The attribute table now has every polygon classified by both its soil type (e.g., Group B) and land use (e.g., Agricultural), allowing you to assign specific curve numbers to each unique plot.

D. Dissolve

Exam Example: You receive a highly detailed geological map with 50 different sub-classifications of soil types. For your preliminary analysis, you only need to differentiate between "Coarse-Grained" and "Fine-Grained" soils. You add a new attribute classifying them broadly, and use the Dissolve tool based on that new attribute. The tool erases the boundaries between adjacent polygons of the same broad category, merging hundreds of small polygons into just two massive geological zones.

E. Buffer

Exam Example: You are tasked with environmental compliance for a new road construction project. Regulations state that no heavy machinery can operate within 100 meters of a natural water body. You use the Buffer tool on the "Rivers and Lakes" vector layer, setting a distance of 100m. This generates a new polygon representing the restricted exclusion zone along the entire water network.

F. Merge

Exam Example: You are analyzing traffic volume across a large region, but the road network shapefiles were provided by two different municipal governments. You use the Merge tool to combine the "Municipality A Roads" and "Municipality B Roads" into a single, continuous transportation dataset, stacking their attribute tables together so you can run a unified spatial analysis.

G. Erase

Exam Example: You are finding a suitable location for a new landfill. You have a polygon layer of "Available Government Land." However, you must avoid environmentally sensitive areas. You use the Erase tool, inputting the government land and using a "Protected Forests and Watersheds" layer as the erase feature. The output is the available land with the protected areas digitally "punched out" or removed.

H. Map Join / Spatial Join

Exam Example: You have a point layer representing hundreds of individual California Bearing Ratio (CBR) soil test locations along a proposed highway alignment, and a polygon layer representing different geological wards. A Spatial Join will append the ward information (like ward number and dominant rock type) directly into the attribute table of each individual CBR point based entirely on which polygon the point physically falls inside.

I. Update

Exam Example: You have a master land-use map of a rural municipality from 2020. Recently, a "Smart Village" initiative successfully developed a specific ward, completely changing its infrastructure and land use. You use the Update tool to overlay the new 2026 ward data onto the old 2020 master map. The old map remains intact everywhere else, but the exact footprint of the upgraded ward is completely replaced by the new geometry and attributes.

J. Split

Exam Example: You have a complete, national-level vector layer of every bridge in the country. For administrative distribution, you need to send files to the respective provincial transport offices. You use the Split tool, using a "Provincial Boundaries" polygon layer as the split feature. The tool automatically chops the national bridge layer into seven completely separate shapefiles, one for each province.

1. The Theory Question 2071 Magh, Q7

Question: Define the term spatial analysis. Explain different types of vector overlay operations in GIS. Also explain three examples of neighborhood functions. [2+4+4]

A. Definition of Spatial Analysis (2 Marks)

Spatial analysis is the core analytical process in GIS where we extract or create new information about a set of geographic features. It involves evaluating suitability, predicting outcomes, measuring proximities, and modeling relationships between spatial data to solve complex engineering and planning problems.

B. Types of Vector Overlay Operations (4 Marks)

Vector overlay involves combining two or more separate polygon layers to create a completely new layer containing both the spatial boundaries and the attribute data of the inputs.

Intersect (The "AND" Operator): Keeps only the overlapping regions of the input layers. The output attribute table combines data from both.

Union (The "OR" Operator): Keeps the entire extent of all input layers. Polygons are split wherever lines cross, and the attribute table joins all data.

Identity / Erase: Identity keeps the base map's boundaries but adds the overlay's attributes. Erase completely removes the overlapping portions from the base map.

C. Three Examples of Neighborhood Functions (4 Marks)

Neighborhood functions analyze the relationship between a target feature and its surrounding area.

  1. Buffering: Creating a zone of a specified distance around a point, line, or polygon (e.g., establishing a 100m no-build zone around a river).
  2. Near / Distance Analysis: Calculating the exact shortest distance from one feature to another (e.g., finding the distance from a proposed school to the nearest existing hospital).
  3. Thiessen Polygons (Voronoi Diagrams): Creating boundaries around a set of points so that any location inside a given polygon is closer to that polygon's central point than to any other point (e.g., mapping the service areas for different rainfall stations).

Landfill Site Selection Workflow 10 Marks

Question: Write about the necessary tools and explain how you will perform the following analyses using GIS: Best location for landfill area so that following constraints are fulfilled: Houses > 1000m, River > 1500m, Road < 500m, Water bodies > 1500m. Provided data: Shapefile of river, land use data, point data of the houses, shapefile of roads, polygon shapefile of the study area. [10]

Here is the precise step-by-step workflow you need to write:

Step 1: Buffer the Restricted Features (Tool: Buffer)

Since a landfill is hazardous, it must be far from people and water. We buffer the restrictions:

  • Buffer the "Houses" point layer by 1,000m.
  • Buffer the "River" line layer by 1,500m.
  • Buffer the "Water bodies" polygon layer by 1,500m.

Step 2: Combine the Restricted Zones (Tool: Union or Merge)

Use the Union tool to combine the three buffers from Step 1 into a single, massive "Restricted Area" polygon layer.

Step 3: Extract the Available Land (Tool: Erase)

Take the total "Study Area" polygon as the input feature, and use the "Restricted Area" from Step 2 as the erase feature. The output is a map of the "Available Land" where it is legally safe to build.

Step 4: Buffer the Favorable Proximity Feature (Tool: Buffer)

The site must be accessible for garbage trucks. Buffer the "Road" line shapefile by 500m to create the "Accessibility Zone".

Step 5: Isolate the Final Suitable Sites (Tool: Intersect)

Intersect the "Available Land" (from Step 3) with the 500m "Road Buffer" (from Step 4). The resulting polygons are the exact locations that meet every single distance constraint.

Step 6: Filter by Appropriate Land Use (Tool: Select by Attributes)

Using the provided "Land use data" shapefile, perform a Spatial Join with your final sites. Then, run an SQL query (e.g., SELECT * WHERE Land_Use = 'Barren') to ensure the landfill isn't placed on top of agricultural or forested land.
