Chapter 7

Spatial Analysis

Part 1: Concept, Purpose, and DEM Generation Covers: 2071 Q9, 2072 Q9, 2079 Q5b, 2081

1. What is Spatial Interpolation?

Spatial interpolation is the mathematical and statistical process of estimating the values of a continuous geographic phenomenon at unknown, unsampled locations based on the known values of surrounding sampled points.

2. What is the purpose of spatial interpolation?

In reality, it is impossible, physically and financially, to measure data (like elevation, rainfall, or soil strength) at every single square meter of a study area. The primary purpose of interpolation is to take isolated sample points (like weather stations, borehole logs, or GPS spot heights) and create a continuous surface (a raster grid) that models the entire region.

3. Why is interpolation needed for producing a DEM?

A Digital Elevation Model (DEM) is a continuous raster grid where every single pixel contains an elevation value. However, field surveyors and LiDAR point clouds only capture elevation at discrete, scattered points. Interpolation is absolutely strictly required to calculate the elevation of the "empty" spaces between the surveyed points, effectively creating a solid, continuous topographic surface from isolated data.

Part 2: Classification and Types of Interpolation Covers: 2071 Q9, 2074 Q3a, 2075 Q5a

If asked to classify or explain the different types, break them down into two main families:

A. Deterministic Methods

These use predefined mathematical equations based on the distance or geometric layout of the known points. They do not account for statistical probability.

  • Inverse Distance Weighted (IDW): Assigns values to unknown points by using a weighted average of the surrounding known points. The closer a known point is, the more heavily it influences the unknown value.
  • Spline (Minimum Curvature): Fits a mathematically smooth, flexible "rubber sheet" through all the known data points, minimizing the overall curvature of the surface.
  • Thiessen Polygons (Nearest Neighbor): Creates sharp boundaries. Any unknown location simply takes the exact value of the single nearest known point.

B. Geostatistical Methods

These rely on both the distance and the statistical relationship (spatial autocorrelation) among the measured points.

  • Kriging: A highly advanced method that not only estimates the unknown values but also provides a map of the estimated error/uncertainty. It looks at the overall spatial trend and variance in the dataset.

Part 3: Detailed Method Comparison (IDW vs. Spline) Covers: 2071 Q9, 2073 Q11c, 2081

The examiners frequently ask you to describe one method in detail, discuss its pros and cons, or compare IDW and Spline directly.

Inverse Distance Weighted (IDW)

  • Concept: Assumes that "things that are closer are more alike."
  • Advantages: Very simple to understand, fast to compute, and respects the exact values of the sampled points.
  • Disadvantages: It cannot predict a value higher than the highest known point or lower than the lowest known point (no extrapolation). It often creates artificial "bullseye" or circular patterns around isolated data points.

Spline (Minimum Curvature)

  • Concept: Fits a smooth curve through the points, much like bending a flexible ruler.
  • Advantages: Creates highly visually appealing, smooth surfaces. Unlike IDW, a Spline can predict values outside the range of the input data (e.g., predicting a mountain peak even if surveyors only measured points on the slopes).
  • Disadvantages: Highly sensitive to outliers. If one data point is incorrectly recorded as extremely high, the Spline will wildly distort the surrounding surface to force the "rubber sheet" through that error.

Part 4: Engineering Applications Covers: 2074 Q3a, 2079 Q5b

To answer "application in different engineering fields" effectively, provide these specific examples:

  1. Hydrology (mHM Modeling): Interpolating isolated daily rainfall data from a few regional meteorological stations to create a continuous precipitation raster across an entire river basin.
  2. Geotechnical Engineering (Site Response Analysis): Interpolating shear wave velocity (Vs30) or bedrock depth from scattered borehole logs to create a continuous subsurface vulnerability map for the Kathmandu Valley.
  3. Water Resources (HEC-RAS): Interpolating riverbed bathymetry from surveyed cross-sections to generate a continuous 3D channel terrain for 2D flood inundation modeling.
  4. Environmental Engineering: Modeling the dispersion plume of airborne pollutants or groundwater contamination spreading from an industrial site based on a few scattered air/water quality monitoring sensors.

7.2 Raster Analysis Including Topological Overlay

Part 1: Theoretical Foundations of Raster Analysis Covers: 2071 Magh Q8, 2072 Ashwin Q3a

1. Raster-Based Analysis: Advantages and Disadvantages

To secure full marks on this recurring theory question, list these exact points:

  • Advantages:
    • The data structure is a simple matrix of cells, making mathematical modeling (Map Algebra) extremely fast and computationally efficient.
    • It is the absolute best format for representing continuous surfaces that change gradually, such as elevation (DEMs), rainfall distributions, or soil moisture content.
    • Cell-to-cell overlay operations are perfectly aligned, eliminating the complex geometry calculations required in vector overlay.
  • Disadvantages:
    • It suffers from the "stair-step" or pixelated effect, meaning it cannot precisely represent sharp boundaries (like a property line or a strict building footprint).
    • High-resolution grids (e.g., a 1m x 1m DEM) create massive file sizes that require significant storage and processing power.
    • It is inefficient for network analysis (like calculating traffic volume or routing).

2. Map Overlay vs. Map Calculations

  • Map Overlay (Vector Focus): Involves physically calculating the geometric intersections of polygons, lines, or points. It alters the spatial boundaries to create new shapes (e.g., cutting a land-use polygon with a flood-zone polygon).
  • Map Calculations (Raster Focus): Also known as Map Algebra. It does not change geometry because the grid structure is fixed. Instead, it stacks identical grids on top of each other and performs pixel-by-pixel mathematical operations (e.g., taking a raster of January rainfall and a raster of February rainfall, and simply adding the overlapping pixel values to get a Total Rainfall raster).

Part 2: The Four Tiers of Raster Operations Covers: 2080 Chaitra Q8b, 2075 Magh Q5b, 2073 Magh Q11

Examiners love asking about how raster tools are categorized. Raster functions are defined by how many cells the algorithm looks at to calculate a new value.

1. Local Functions (Cell-by-Cell)

  • Concept: The value of a cell in the output raster is calculated using only the value of the exact same cell location in the input raster(s). It does not care about neighboring pixels.
  • The 2080 Chaitra Question: Why is NDVI a local function?
    Normalized Difference Vegetation Index (NDVI) is calculated using the formula (NIR - Red) / (NIR + Red). To find the NDVI of a specific 30m × 30m patch of land, the software only needs the Near-Infrared and Red band values of that exact single pixel. It performs the math purely vertically through the stacked layers.

2. Focal / Neighborhood Functions

  • Concept: The output value of a cell is calculated based on the values of the cells immediately surrounding it (typically a 3 × 3 moving window).
  • The 2080 Chaitra Question: Why are Slope and Aspect neighborhood functions?
    You cannot calculate a slope from a single isolated elevation point. To determine how steep a terrain is (Slope) or which direction it faces (Aspect), the algorithm must look at the central cell and compare its elevation to the 8 surrounding neighbor cells to calculate the rate of change.

3. Zonal Functions (Zonal Statistics)

  • Concept: Calculates statistical values (Mean, Sum, Max, Min) for raster cells that fall within the boundaries of a defined "zone." A zone can be defined by another raster or a vector polygon layer.
  • Application Example: You are conducting Site Response Analysis for your final year project. You have a continuous raster grid of shear wave velocities (Vs30) for the Kathmandu Valley, and a vector polygon layer of municipal wards. By running Zonal Statistics, the software will calculate the average Vs30 value specifically within each ward's boundaries, outputting a table summarizing the seismic vulnerability per ward.

4. Global Functions

  • Concept: The output value of each cell is potentially a function of all the cells in the entire input raster.
  • Example: Euclidean Distance, where the distance from a source is calculated across the entire extent of the map.

Part 3: Deep Dive into Specific Raster Tools (8 to 16 Mark Questions) Covers: 2073, 2074, 2076, 2078, 2079, 2082

When asked to explain and illustrate these tools, define the tool and provide a concrete engineering application.

1. Slope and Aspect

  • Slope: Calculates the maximum rate of change in elevation over distance (steepness), outputting values in degrees or percent rise.
    • Application: When modeling the application of machine learning for predicting cost overruns in road construction, you use a slope raster to identify extreme terrains where excavation costs will mathematically multiply.
  • Aspect: Calculates the compass direction (0° to 360°) that the downward slope faces.
    • Application: Identifying south-facing mountain slopes in rural villages to maximize the efficiency of solar panel installations.

2. Hillshade and Viewshed

  • Hillshade: Simulates the interaction of sunlight with terrain (requiring a specific sun azimuth and altitude) to create a 3D, shaded relief effect purely for visualization and cartography.
  • Viewshed: Determines exactly which raster cells are visible from one or more observer points, accounting for the blocking effect of terrain.
    • Application: Determining if a new telecommunication tower placed on a hill in Bhadaure will actually have a direct line-of-sight to the surrounding settlement, or if a ridge blocks the signal.

3. Euclidean Distance and Cost Path

  • Euclidean Distance: Creates a raster where every single pixel's value represents its straight-line distance to the nearest source feature (like a river or a road).
    • Application: When designing a toilet for a rural sanitation initiative, you create a Euclidean distance raster away from all groundwater wells. You then mask out any pixels with a value less than 30 meters to strictly follow contamination regulations.
  • Cost Path: Determines the most cost-effective or "path of least resistance" route between a source and destination, taking into account a "cost surface" (friction like steep slopes, rivers, or dense forests).
    • Application: Modeling an optimized, physically least-strenuous new trekking route connecting the Mardi Himal trail to a neighboring valley by avoiding slopes greater than 30 degrees.

4. Reclassify and Map Algebra

  • Reclassify: Reassigns ranges of continuous raster values into simplified, discrete integer classes.
    • Application: Taking a continuous Slope raster (0 to 90 degrees) and reclassifying it into a vulnerability index: 0-15° = 1 (Safe), 15-30° = 2 (Moderate risk), >30° = 3 (High landslide risk).
  • Map Algebra: The fundamental mathematical language of raster analysis, allowing you to add, subtract, multiply, or divide entire raster grids against each other.

Additional Raster Operations

1. Local Operations (Mean, Sum, Max, Min) Asked in 2073 Magh Q11

  • Concept: These are technically known as "Cell Statistics." A mathematical operation (like calculating the mean or sum) is performed on a cell-by-cell basis across multiple overlapping input raster layers. The output cell at coordinates (X,Y) is the sum or mean of all the (X,Y) cells in the input stack.
  • Mechanism: If you stack three rasters, the software drills straight down through pixel (Row 1, Column 1) in all three layers, calculates the average, and writes that single average value into pixel (Row 1, Column 1) of the new output raster.
  • Exam Example: When preparing meteorological data for the mHM (mesoscale Hydrologic Model), you have 12 separate raster grids, each representing the total rainfall for a specific month. To find the annual precipitation, you use the Local Sum operation. The software stacks the 12 grids and adds the pixel values together vertically, outputting a single "Total Annual Rainfall" raster grid.

2. Summarize Zone (Zonal Statistics) Asked in 2075 Magh Q5b, 2076 Bhadra Q7a, 2073 Magh Q11

  • Concept: This function calculates summary statistics (like Area, Sum, Mean, Maximum, Minimum) for a continuous "Value Raster" strictly within the boundaries defined by a separate "Zone Dataset". The Zone Dataset can be either a vector polygon shapefile or an integer raster.
  • Mechanism: The software identifies all the pixels of the Value Raster that physically fall inside Zone A, lumps them together, calculates the requested statistic (e.g., the average), and then moves on to Zone B.
  • Exam Example: For your final year project on Site Response Analysis, you have a highly detailed, continuous raster grid showing Peak Ground Acceleration (PGA) across the Kathmandu Valley. You also have a polygon shapefile of the municipal wards. Using the Summarize Zone tool, the software will calculate and output a table showing the exact Mean and Maximum PGA specifically for each individual ward, allowing you to rank the wards by seismic vulnerability.

3. Proximity Map Calculation & Distance Asked in 2075 Magh Q5b, 2076 Bhadra Q7a

Distance and Proximity are often used interchangeably in raster analysis, but they represent slightly different ways of modeling "how close" something is.

Distance (Euclidean Distance)

  • Concept: Calculates the straight-line distance from every single cell center to the closest source feature.
  • Mechanism: It generates a continuous raster where the numerical value of every single pixel represents its distance (in meters or feet) to the target.
  • Exam Example: When mapping the "Smart Village" initiative, you need a general overview of water accessibility. You run a Distance function on the "Water Tap" point layer. The output is a gradient raster where pixels right next to the taps have values near 0m, and pixels on the outskirts of the village have values of 500m or more.

Proximity (Raster Buffering / Allocation)

  • Concept: Instead of a continuous distance gradient, a Proximity calculation often groups cells into distinct bands or allocates them to the nearest source.
  • Mechanism: It classifies pixels based on a threshold. For example, assigning a value of "1" to all pixels within 30 meters of a source, and "0" to everything else (acting like a vector buffer, but in grid format).
  • Exam Example: To enforce sanitation regulations for a new rural toilet, it must be located at least 30 meters away from any drinking water well. You run a Proximity map calculation on the wells, instructing the tool to flag all raster cells within a 30m radius. This instantly generates a restricted "no-build" pixel zone across your study area.

2. The Reservoir Site Selection Scenario 2081 Chaitra – 10 Marks

Question: You are tasked with identifying suitable locations for building a new reservoir in a region. The ideal location must meet the following criteria:

  1. The area must have a slope of less than 10 degrees to ensure stability.
  2. It must be within 1 kilometer of a river to allow for water supply.
  3. The land cover must be non-forested (e.g., grassland or bare soil) to minimize environmental impact.
  4. The area must have a high water-retention capacity, indicated by soil types classified as "clay" or "silt".

The data provided are DEM, Soil type, River, and Land cover data. Using raster data analysis, describe the steps you would take to identify suitable locations. Include at least four raster analysis functions in your explanation.

Approach: This is a classic Boolean Overlay (True/False) suitability model. Each dataset is independently processed, reclassified into binary (1=Suitable, 0=Unsuitable), and then multiplied cell-by-cell. The multiplication acts as a logical AND — a cell must pass all four tests to be marked suitable.

The Four Raster Analysis Functions Used

  1. Slope (from DEM): A Focal/Neighborhood function that computes the rate of elevation change using a 3×3 moving window. Input: DEM raster. Output: Continuous slope raster in degrees (0°–90°). This is a neighborhood function because each pixel's slope depends on its 8 surrounding neighbors.
  2. Euclidean Distance (from River): A Distance function that calculates straight-line distance from every cell center to the nearest river feature. Input: River vector (or raster). Output: Continuous distance raster in meters. Cells near rivers have low values; distant cells have high values.
  3. Reclassify: A Local function that reassigns ranges of continuous values into discrete binary classes. Applied independently to all four datasets:
    • Slope: 0°–10° → 1, >10° → 0
    • Distance: 0–1000 m → 1, >1000 m → 0
    • Land Cover: Grassland/Bare soil → 1, Forest → 0
    • Soil Type: Clay/Silt → 1, Sand/Gravel → 0
  4. Raster Calculator / Map Algebra: A Local function that performs cell-by-cell arithmetic across multiple aligned rasters. Here we multiply all four binary grids: Final = Slope_Rec × Distance_Rec × LandCover_Rec × Soil_Rec. Any cell with a 0 in any layer becomes 0 in the output (Boolean AND logic).

Step 1: Calculate Slope (Tool: Slope)

Step 2: Calculate Proximity (Tool: Euclidean Distance)

Input the River vector data and run the Euclidean Distance tool to create a raster where every pixel value represents its distance from the river.

Step 3: Standardize the Criteria (Tool: Reclassify)

You must convert all datasets into a binary scale where 1 = Suitable and 0 = Unsuitable.

  • Reclassify Slope: 0° to 10° = 1. Greater than 10° = 0.
  • Reclassify Distance: 0 to 1000m = 1. Greater than 1000m = 0.
  • Reclassify Land Cover: Grassland/Bare soil = 1. Forested = 0.
  • Reclassify Soil: Clay/Silt = 1. Sand/Gravel = 0.
