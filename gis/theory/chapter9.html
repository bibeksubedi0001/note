1. The Foundation: Watersheds and the DEM

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Define watershed and explain filled DEM and flow direction [1+5] (2072 Magh, Q10)

Definition of a Watershed:

A watershed (also known as a catchment area or drainage basin) is a topographically defined geographical area of land where all surface water (from rain, melting snow, or ice) converges and drains off into a single, common outlet point, usually known as a "pour point" (like a river mouth, a lake, or a dam). In GIS, the boundary of a watershed is defined by topographic ridges (drainage divides) extracted from a Digital Elevation Model (DEM).

2. Pre-processing the DEM: Sinks and Filled DEMs

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Define watershed and explain filled DEM and flow direction [1+5] (2072 Magh, Q10)
  2. Explain how a sink is developed in DEM and how it can be removed [4] (2074 Bhadra, Q2a)
  3. Explain sinks and filled DEM (2075 Magh, Q3a)

Before any hydrological analysis can begin the raw DEM must be checked for errors.

What is a Sink?

A sink (or depression) is a cell or a group of adjacent cells in a DEM that are completely surrounded by cells with higher elevation values. Because water flows downhill by gravity any water that enters a sink is trapped and cannot flow out causing the digital river network to artificially disconnect and stop.

How is a Sink Developed in a DEM?

  1. Data Errors (Artifacts): Most sinks are artificial errors created during the data collection process (e.g. radar shadow in SRTM or ASTER data) or during the spatial interpolation of the DEM grid.
  2. Resolution Limitations: If the grid resolution is too coarse (e.g. 90m pixels) a narrow gorge might be missed making it look like a closed basin.
  3. Natural Sinks: Occasionally they are real geographical features such as glacial hollows quarries or karst (limestone) sinkholes.

How is it Removed? (The "Fill" Operation)

Sinks are removed using the Fill Sink geoprocessing tool. The software algorithm identifies the trapped cells and artificially raises their elevation values until they are perfectly level with the lowest surrounding boundary cell (the "pour point" of the sink).

What is a Filled DEM?

A Filled DEM (or Depressionless DEM) is the final output of the Fill operation. It is a corrected elevation surface where all artificial sinks have been eliminated ensuring that water can flow continuously across the landscape to the edges of the map.

3. Core Hydrological Algorithms

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Explain Sink, Flow Direction, Flow Accumulation, River Grid, and stream lines [5] (2074 Magh, Q5a)

Once you have a Filled DEM you run a sequence of specific tools to create the rivers.

A. Flow Direction

Flow Direction calculates the exact path water will take as it moves from one pixel to the next. The standard method used is the D8 (Eight-Direction) Algorithm. The software evaluates the center cell and its 8 immediate neighbors. It calculates the drop in elevation and assigns a flow direction code pointing to the single neighboring cell that has the steepest downward slope.

B. Flow Accumulation

Using the flow direction network the Flow Accumulation tool calculates how much water flows into every single cell. It does this by counting the total number of upstream cells that pour into a specific downslope cell.

  • A cell on a mountain ridge will have an accumulation of 0 (no water flows into it).
  • A cell at the bottom of a major valley will have a massive accumulation number (thousands of cells are draining into it).

C. River Grid (Stream Definition)

Not every cell with flow accumulation is a river; some are just muddy slopes. To define an actual river you apply a Threshold using Map Algebra. For example you might set a rule: If Flow Accumulation > 500 cells then it is a River (1) else it is Not a River (0). The output is a binary River Grid where the pixels representing streams are highlighted.

D. Stream Lines (Stream to Feature)

The River Grid is still a raster (pixels). To make it useful for mapping it is converted into vector polylines using the "Stream to Feature" tool. These resulting vector lines are the Stream Lines.

Once stream lines are created hydrologists classify their size and importance using stream ordering. While the Strahler method is the most common the exams specifically ask for the Shreve Method.

The Shreve Method:

In the Shreve system all exterior links (the smallest starting streams with no tributaries) are assigned a magnitude of 1.

Unlike other methods when two streams intersect their magnitudes are strictly added together.

  • If a magnitude 1 stream joins another magnitude 1 stream the resulting downstream link is 2 (1+1).
  • If a magnitude 2 stream joins a magnitude 3 stream the resulting downstream link is 5 (2+3).
  • Advantage: Because it is purely additive the magnitude of a downstream link perfectly represents the total number of starting tributaries contributing to that point making it excellent for estimating total water discharge.

5. The Complete Workflow / Scenario Question

This is a comprehensive 10-mark question that requires you to outline the entire process including initial data management.

Question Workflow: Generating a River Network from Multiple DEMs

Step 1: Data Preparation (Mosaic)

Since multiple DEMs cover the area the first step is to use the Mosaic to New Raster tool. This stitches the multiple separate DEM grids together into one large continuous regional DEM.

Step 2: Area Extraction (Clip/Extract by Mask)

Use the provided "Polygon Vector Data" (the study area) as a mask. Use the Extract by Mask (or Clip) tool to cut the large mosaicked DEM down to the exact boundary of your study area to save processing time.

Step 3: Remove Depressions (Fill)

Run the Fill tool on the clipped DEM to remove any sinks and create a depressionless DEM.

Step 4: Calculate Flow Direction

Input the Filled DEM into the Flow Direction tool. This applies the D8 algorithm to determine the downslope path for every pixel.

Step 5: Calculate Flow Accumulation

Input the Flow Direction raster into the Flow Accumulation tool. This calculates the total number of upstream cells contributing to each pixel.

Step 6: Stream Definition (Map Algebra/Raster Calculator)

Apply a threshold to define the rivers. Use the Raster Calculator with a conditional statement: Con("Flow_Accumulation" > Threshold_Value 1). This outputs a binary River Grid.

Step 7: Convert to Vector (Stream to Feature)

Finally use the Stream to Feature tool (inputting both the binary River Grid and the Flow Direction raster). This converts the raster pixels into smooth vector polylines resulting in your final continuous River Network shapefile.

Chapter Summary

This chapter covers the GIS-based process of extracting river networks and watershed boundaries from Digital Elevation Models (DEMs). The process follows a systematic sequence: first, flow direction is determined using the D8 (deterministic eight-neighbor) algorithm, which assigns each cell a flow direction toward the steepest downslope neighbor. Next, flow accumulation counts the number of upstream cells draining through each cell — cells with high accumulation values represent stream channels. A threshold value is applied to the flow accumulation grid to extract the river network (cells above the threshold are classified as streams). Finally, watershed boundary delineation identifies the drainage area contributing flow to a specified outlet point by tracing flow directions upstream. The chapter also covers DEM preprocessing steps like filling sinks (depressionless DEM), stream ordering (Strahler and Shreve methods), and pour point identification. These techniques are fundamental for hydrological modeling, flood risk analysis, and water resource management.
