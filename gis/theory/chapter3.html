Key Concepts (Textbook pp.67–86)

  • Geographic Phenomenon: “A manifestation of an entity or process of interest that can be named or described, can be georeferenced, and can be assigned a time interval at which it is/was present.” (Textbook p.67)
  • Geographic Field: A phenomenon for which, at every point in the study area, a value can be determined: f(x,y) → value. Examples: elevation, temperature, rainfall. (Textbook p.69)
  • Geographic Object: Well-distinguished, discrete, bounded entities that populate the study area. The space between objects is potentially “empty.” Examples: buildings, roads, lakes. (Textbook p.70)
  • Rule of Thumb: Natural phenomena are usually modelled as fields (e.g., elevation surface). Man-made features are usually modelled as objects (e.g., buildings, roads). (Textbook p.70)
  • Tobler’s First Law of Geography: “Locations that are closer together are more likely to have similar values than locations that are far apart.” This principle of spatial autocorrelation underpins interpolation and geostatistics. (Textbook p.83)
  • Continuous vs. Discrete Fields: Continuous fields change gradually (elevation, temperature). Discrete fields divide space into bounded parts, each with a single value (land use, soil type). (Textbook pp.72–73)
  • Crisp vs. Fuzzy Boundaries: Crisp boundaries can be determined with arbitrary precision (man-made: political borders, property lines). Fuzzy boundaries are transition zones (natural: soil types, vegetation gradients). (Textbook p.81)

1. Types of Spatial Data Models in GIS

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Explain different types of data models [4] (2073 Bhadra, Q3a)
  2. Explain different types of data models [3] (2076 Bhadra, Q3a)
  3. Explain different types of data models in GIS [6] (2078 Chaitra, Q1)

A data model in GIS is a mathematical and conceptual framework used to represent real-world geographic features in a digital format. The primary spatial data models used to abstract and manage this information include:

  • Vector Data Model: Represents discrete geographic features using coordinates to form points, lines, and polygons. It is ideal for objects with distinct boundaries, such as buildings, administrative borders, or road networks.
  • Raster Data Model: Represents continuous spatial phenomena through a grid of equally sized cells (pixels), where each cell holds a specific value. It is best suited for continuous data like elevation, temperature, or satellite imagery.
  • TIN (Triangulated Irregular Network): A vector-based representation of a continuous surface consisting of non-overlapping, contiguous triangles. It is primarily used for high-precision elevation modeling and terrain analysis.

2. The Network Data Model

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. What is the Network data model? Describe with examples [Not specified] (2075 Bhadra, Q5)

The Network Data Model is a specialized, topologically structured vector model designed to represent interconnected linear features. It defines how lines connect to each other at nodes (intersections) to model the flow of resources or movement.

  • Examples: It is heavily used in civil engineering and urban planning to map road networks (for finding the shortest route), river systems (for tracing hydrological flow), and utility grids (such as water pipes or electrical lines to manage distribution and outages).

4. Levels of Measurement and their Impact on Symbology

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. What are the levels of measurement of attribute data? How do they affect the symbology? [6] (2079 Chaitra, Q3a)
  2. Explain four levels (nominal, ordinal, interval, ratio) of commonly recognized data measurement (2081 Chaitra)

The scale of measurement of attribute data dictates the analytical operations possible and directly determines the symbology used to visually communicate the map's message.

  • Nominal Data: Categorical data with no inherent order or ranking (e.g., land use types like forest, agriculture, urban).
    • Symbology: Represented using "Unique Values." Different categories are shown using distinct, unranked colors or distinct symbols (e.g., green for forest, gray for urban).
  • Ordinal Data: Categorical data that has a logical order or rank, but the intervals between ranks are not mathematically equal (e.g., risk zones classified as low, medium, and high).
    • Symbology: Represented using "Graduated Colors," typically a sequential color ramp (e.g., light yellow for low risk, transitioning to dark red for high risk).
  • Interval Data: Numeric data with a known, equal interval between values, but lacking a true zero point (e.g., Temperature in Celsius or elevation above sea level, where zero does not signify the absolute absence of the quantity).
    • Symbology: Often mapped using classified numeric ranges with graduated color schemes, or diverging color ramps to show variation above and below a central mean.
  • Ratio Data: Numeric data with a true, absolute zero point, allowing for proportional comparisons (e.g., population count, area, distance).
    • Symbology: Effectively represented using "Proportional Symbols" (e.g., larger circles for cities with higher populations) or "Graduated Colors" normalized by area (like a choropleth map showing population density).

1. Vector vs. Raster Data Models

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Explain Vector model and Raster GIS Model with suitable examples [Part of 10] (2074 Magh, Q1)
  2. Discuss the key differences between vector and raster data models, highlighting strengths and weaknesses. Include a sketch [3+4] (2080 Chaitra, Q1)
  3. What are the basic elements of raster data model? Explain advantage and disadvantage of raster vs vector [3+] (2072 Magh, Q3)

Geographic Information Systems (GIS) use two main ways to represent the real world: Vector and Raster.

Feature Vector Data Model Raster Data Model
Basic Concept Uses geometry (points, lines, polygons) to represent exact real-world shapes. Uses a grid of square cells (pixels) to cover an area, where each cell has a value.
Best Used For Discrete data with clear boundaries (e.g., a specific building, a road, a property line). Continuous data that changes smoothly over space (e.g., elevation, temperature, rainfall).
Visualizing a Scene A tree is a single point, a road is a smooth line, and a building is an exact enclosed shape (polygon). The entire area is a grid. A building is a group of cells colored "building," and a road is a jagged line of "road" cells.
Advantages Very accurate boundaries; smaller file sizes; excellent for network analysis (like finding the shortest driving route). Simple structure; great for overlaying maps; handles continuous surfaces and satellite imagery perfectly.
Disadvantages Complex data structure; bad for continuous gradients like elevation; overlaying maps requires heavy computing. Can look pixelated/blocky; large file sizes; terrible for precise network routing.

Important Distinctions (Textbook pp.86–94)

  • Grid vs. Raster: These terms are not synonymous. A grid stores values at the intersections of regularly spaced lines. A raster stores values for the entire cell area (cell values, not point values). The value for a raster cell is assumed valid for all locations within that cell. (Textbook p.86)
  • Resolution: The cell size determines the level of detail. Smaller cells → better quality but larger file size. Cell size should match the accuracy of the original data source. (Lectures p.13)
  • Raster Algebra: Because raster cells can be handled as 2D arrays, many analytical operations (overlay, neighbourhood) are easy to program. This makes raster structures popular for mathematical modelling. (Lectures p.13)

2. Types of Raster Models

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Explain different types of Raster Models [6] (2071 Bhadra, Q3a)

Raster models are grouped into three basic types based on what their grid cells represent:

  • Continuous (Surface) Rasters: Values change gradually. For example, an elevation map where cell values represent height above sea level. There are no sudden boundaries.
  • Discrete (Thematic) Rasters: Values represent categories. For example, a land-use map where all cells representing "forest" have a value of 1, and all cells representing "water" have a value of 2.
  • Image Rasters: Values represent light or energy. This is typical for satellite imagery or drone photos.

3. Raster Compression & Vectorization

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Explain the raster compression techniques with appropriate examples [4] (2079 Chaitra, Q3b/Q4b)
  2. Define the terms region quadtree and vectorization [1+1] (2071 Magh, Q2a)

Raster files (grids) can get huge. We use compression techniques to make the files smaller without losing information.

  • Run Length Encoding (RLE):
    • How it works: Instead of saving the value of every single cell in a row, it groups identical values together.
    • Example: If a row has five cells of "Forest" (Value 1), instead of storing 1, 1, 1, 1, 1, RLE stores it as (5 of 1).
  • Region Quadtree:
    • How it works: It divides the entire map grid into four large square blocks (quadrants). If a block is completely uniform (e.g., all water), it stops. If a block has mixed values (e.g., water and land), it splits that block into four smaller blocks. It repeats this until every block contains only one uniform value.
  • Vectorization (Concept):
    • How it works: This is the process of converting a Raster grid into Vector lines and polygons. Think of it as tracing a scanned pixelated image to create smooth, editable lines.

4. Raster Data Storage Formats (BSQ, BIL, BIP)

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Illustrate BSQ, BIL, and BIP raster format (2075 Magh, Q5d)

When working with satellite images that have multiple layers (like a Red band, a Green band, and a Blue band), the computer needs a rule for how to store that data line-by-line.

  • BSQ (Band Sequential): Stores all the data for Band 1 first, then all the data for Band 2, and so on. (Great if you only want to look at one band at a time).
  • BIL (Band Interleaved by Line): Stores row 1 of Band 1, then row 1 of Band 2, then row 1 of Band 3. Then it moves to row 2. (A good middle-ground format).
  • BIP (Band Interleaved by Pixel): Stores pixel 1 of Band 1, pixel 1 of Band 2, pixel 1 of Band 3. Then moves to pixel 2. (Great for analyzing the exact color of a specific spot quickly).

3.4 Indexing and Hierarchical Data Structures

No direct past exam questions have been asked from this section, but it is part of the syllabus and may appear in future exams.

When a GIS database contains millions of spatial features (e.g., every building in Kathmandu Valley), finding which features fall within a query area requires an efficient search strategy. Spatial indexing organizes features so the system doesn't have to check every single feature — it can quickly narrow down candidates.

Why Spatial Indexing is Needed

Without indexing, a spatial query like "Find all buildings within 1 km of this road" would require checking every building in the entire database — a brute-force approach that becomes impossibly slow with large datasets. Spatial indexes partition space into manageable regions so only relevant features are examined.

Common Spatial Indexing Methods

  1. Grid Index: The study area is divided into a regular grid of equal-sized cells. Each feature is assigned to the cell(s) it overlaps. During a query, only features in the relevant cells are checked.
    • Advantage: Simple to implement
    • Disadvantage: Inefficient when features are unevenly distributed (many empty cells, some overloaded cells)
  2. Quadtree: A hierarchical structure that recursively subdivides space into four quadrants. Areas with more features are subdivided further, while empty areas remain as large, undivided blocks. (This is related to the quadtree compression technique covered in Section 3.3.)
    • Advantage: Adapts to data density — more detail where needed
    • Disadvantage: More complex to build and maintain
  3. R-Tree (Rectangle Tree): Groups nearby spatial objects into minimum bounding rectangles (MBRs) and organizes them in a balanced tree structure. When searching, the algorithm first checks which bounding rectangles intersect the query window, then examines the actual features only within matching rectangles.
    • Advantage: Very efficient for both point and polygon data; used in most modern GIS (PostGIS, ArcGIS)
    • Disadvantage: MBRs can overlap, causing some redundant checks

Hierarchical Data Structures

In the context of GIS, hierarchical structures organize spatial data in a tree-like fashion where:

  • The root represents the entire study area
  • Each branch represents a spatial subdivision
  • The leaves contain the actual data values or feature references

Examples include the quadtree (divides into 4), octree (divides into 8, used for 3D data), and k-d trees (binary space partitioning). These structures enable fast spatial searches with time complexity of O(log n) instead of O(n).

1. Concept and Importance of Topology

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Write short notes on: Topology [3] (2071 Magh, Q6b)
  2. Why is topology important in GIS? (2075 Bhadra, Q3b)

What is Topology? Topology is a set of rules and mathematical concepts that define the spatial relationships between adjacent or connected geographic features (points, lines, and polygons) in a vector data model. It dictates how features share geometry.

Why is it important in GIS?

  • Data Quality and Validation: Topology ensures data integrity by preventing common digitizing errors. It forces boundaries to match perfectly, eliminating overlapping polygons or empty gaps (slivers) between adjacent features like land parcels.
  • Connectivity (Network Analysis): It explicitly defines how lines connect at intersections (nodes). This is absolutely critical for network analysis, such as finding the shortest driving route, modeling water flow in pipes, or analyzing traffic.
  • Adjacency and Containment: Topology allows the GIS software to "know" spatial facts without calculating them on the fly. It knows which countries share a border (adjacency) or which cities are located entirely within a specific province (containment).

2. The TIN (Triangulated Irregular Network) Data Model

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. Explain TIN Data model [4] (2072 Ashwin, Q1c)
  2. Does TIN belong to a raster or a vector and why? [4] (2071 Magh, Q2a)

What is a TIN? A TIN is a digital data structure used to represent continuous 3D surfaces most commonly terrain or elevation. It constructs this surface using a network of non-overlapping contiguous triangles. The corners of these triangles are created from irregularly spaced data points (called mass points) where each point contains an X Y and Z (elevation) coordinate.

Does TIN belong to Raster or Vector?

A TIN strictly belongs to the vector data model.

Why? Unlike a raster model which relies on a uniform grid of identical square cells (pixels) to represent a surface a TIN is built entirely from fundamental vector geometries. It uses points (the nodes with XYZ coordinates) connected by lines (the edges) to form polygons (the triangular faces). Because it does not use a continuous pixel grid it is classified as a vector surface model.

Delaunay Triangulation (Textbook p.94)

Delaunay triangulation is a method of constructing triangles from a set of sample points such that the triangles are as equilateral as possible. The key property: the circumscribed circle (circumcircle) of each triangle contains no other anchor/sample point inside it. This maximizes the minimum angle of all triangles, avoiding long, thin “sliver” triangles that produce poor interpolation results.

Thiessen/Voronoi Polygons: The dual of a Delaunay triangulation. Each Voronoi polygon contains all locations closer to a particular sample point than to any other sample point. Together, Delaunay triangulation and Voronoi polygons provide complementary views of spatial proximity.

3. Topology within the TIN Data Model

PAST EXAM QUESTIONS ON THIS TOPIC:

  1. What is topology in TIN data model? [2] (2073 Bhadra, Q3b)

What is topology in a TIN?

In a TIN model topology refers to the explicit mathematical relationships maintained between its components: the nodes edges and triangular faces.

  • The Structure: The TIN database doesn't just draw triangles; it explicitly records which three nodes make up a specific triangle and which triangles share a common edge.
  • The Rule: In a proper TIN topology every edge is shared by exactly two adjacent triangles (unless it is on the outer boundary of the model) and multiple triangles share the same corner node.
  • The Benefit: Because these topological relationships are hardcoded the GIS can rapidly perform complex 3D surface analyses across the terrain such as calculating slope aspect (the direction a slope faces) and line-of-sight visibility without having to recalculate how the points connect every time.

Numerical Procedures

Lossless vs. Lossy Compression (Lectures p.14, 16)

  • Lossless compression: Original data can be precisely reconstructed. Run-length encoding (RLE) and block encoding are lossless. Suitable for DEMs and classified rasters where exact values matter.
  • Lossy compression: Data cannot be fully reconstructed but achieves much higher compression ratios. MrSID (wavelet-based) can compress aerial photographs dramatically but is not suitable for DEMs or analytical rasters.
  • JPEG 2000: Uses wavelet compression similar to MrSID, increasingly used for image compression in geospatial applications.
  • File size examples: A 30 m DEM ≈ 1.1 MB; a 10 m DEM ≈ 9.9 MB. High-resolution rasters require compression for practical storage and web delivery. (Lectures p.14)

1. Run Length Encoding (RLE)

(Applies to questions asking for Run Length Encoding)

This method compresses data by grouping identical adjacent cell values in a single row.

  1. Read Row by Row: Start at the top-left cell of the raster grid and read across the first row.
  2. Count Consecutive Values: Count how many times a specific value repeats sequentially.
  3. Write the Pair: Write down the count followed by the value. (For example if you see five cells with the number "27" write it as (5 27) or 5:27).
  4. Reset at Changes or New Rows: When the number changes start a new count. When you reach the end of a row always start a fresh count for the next row (standard row-by-row RLE).
  5. Final Output: Your answer should be a linear list of these pairs for the entire grid.

2. Quadtree Encoding

(Applies to questions asking for Quad tree encoding)

This is a spatial compression method that divides grids into four quadrants (NW NE SW SE).

  1. Check the Whole Grid: Look at the entire grid (e.g. 8x8). Does every single cell have the exact same value?
    • If yes: The entire grid is just one value. Stop.
    • If no: Proceed to Step 2.
  2. Divide into Four: Split the grid evenly into four quadrants.
  3. Evaluate Each Quadrant: Look at the top-left quadrant (NW) first.
    • Are all cells inside it identical? If yes record that value.
    • If there are mixed values split that specific quadrant into four smaller quadrants.
  4. Repeat (Recurse): Continue this process of evaluating and splitting for the NE SW and SE quadrants. Keep splitting mixed blocks until every block you have contains only a single uniform value.
  5. Final Output: Draw a tree diagram. The top root is the whole grid branching into 4 nodes. Uniform nodes stop (leaf nodes). Mixed nodes branch into 4 more nodes.

3. Block Encoding

(Applies to questions asking for Block encoding)

Instead of grouping rows or dividing by strict quadrants this method looks for the largest possible square blocks of identical values anywhere in the grid.

  1. Find the Largest Squares: Scan the grid for the biggest possible square (e.g. 4x4) where all cells are the exact same value.
  2. Record the Block: Write down three things for that block:
    • Its starting coordinate (usually the row and column of its top-left or bottom-left corner).
    • The size of the square (e.g. 4).
    • The value of the cells inside it.
  3. Move to Smaller Squares: Once you've mapped out the largest squares find the next largest (e.g. 2x2 squares) and record them.
  4. Record Single Cells: Finally any leftover cells that don't form a 2x2 or larger square must be recorded as 1x1 blocks.
  5. Final Output: A list of coordinates sizes and values.

4. Chain Encoding

(Applies to questions asking for Chain encoding)

This method is used to trace the boundary of a specific feature (like a polygon of "forest" cells). It relies on a directional grid (usually 0 to 7 where 0 is East 1 is North-East 2 is North etc.).

  1. Pick a Start Node: Choose an outer boundary pixel of the shape you are tracing. Record its (X Y) coordinate as the starting point.
  2. Trace the Boundary: Move to the next adjacent pixel along the boundary of the shape.
  3. Record the Direction: Write down the directional code (0-7) that corresponds to the direction you just moved.
  4. Loop Back: Continue moving along the edge recording the direction code for every step until you arrive back at your starting pixel.
  5. Final Output: The starting coordinate followed by a sequence of directional numbers.

5. Value Point Encoding

(Applies to questions asking for Value point encoding)

This is basically an index of coordinates for every value.

  1. List Unique Values: Look at the grid and write down every unique value that appears (e.g. Forest Water Settlement).
  2. Find Coordinates: Establish a coordinate system (e.g. top-left cell is 0 0 or 1 1).
  3. Group by Value: For each unique value simply write a list of the (X Y) coordinates of every single cell that contains that value.
  4. Final Output: A grouped list: Value A: (1 1) (1 2) (2 1)...

6. Calculating Percentage Decrease in Size

(Applies to questions asking for % decrease in size)

This proves how effective your compression was.

  1. Calculate Uncompressed Size: Count the total number of cells in the original grid (e.g. an 8x8 grid = 64 cells).
  2. Calculate Compressed Size: Count the number of data elements in your compressed version. (For RLE count the number of data pairs you created. For Quadtree count the total number of leaf nodes in your tree).
  3. Apply the Formula: Use the standard percentage decrease formula:
    Percentage Decrease = ((Uncompressed Size − Compressed Size) / Uncompressed Size) × 100%

    Chapter Summary

    This chapter explains the three fundamental spatial data models used in GIS. The Raster Data Model represents the world as a grid of cells (pixels), each holding a single value — ideal for continuous phenomena like elevation, temperature, and rainfall. Raster compression techniques (run-length encoding, quadtree) reduce storage requirements. The Vector Data Model represents features as points, lines, and polygons with precise coordinates — ideal for discrete features like roads, buildings, and boundaries. Topology (the mathematical rules governing spatial relationships like adjacency, connectivity, containment) is critical for vector data integrity. The TIN (Triangulated Irregular Network) Data Model uses irregularly-spaced sample points connected into non-overlapping triangles (Delaunay triangulation) for efficient terrain surface modeling. Indexing and hierarchical data structures (quadtrees, R-trees) for efficient spatial data retrieval are also covered.
