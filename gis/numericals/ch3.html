Key Formulas & Process

1. Full Raster Encoding

Every cell stores its own value. Total storage = rows × cols values.

Formula: Full Size = rows × columns

This is the baseline against which all compression methods are compared.

2. Run-Length Encoding (RLE)

Scans each row left-to-right. Consecutive identical values are stored as a single (value, count) pair.

Size: RLE Size = Total pairs × 2 (each pair has a value and a count)

Process:

• Read row left to right

• Count consecutive identical values

• Write (value, count) for each run

• Repeat for all rows

3. Block Encoding

Stores rectangular blocks of identical values as (value, r1, c1, r2, c2).

Size: Block Size = Number of blocks × 5

Better for data with large rectangular regions of uniform value.

4. Quadtree Encoding

Recursively divides the grid into 4 equal quadrants: NW, NE, SW, SE.

• If a quadrant is uniform (all same value) → it becomes a leaf node

• If a quadrant is mixed → subdivide into 4 again

• Requires power-of-2 grid size (4×4, 8×8, 16×16, etc.)

Total nodes = internal nodes + leaf nodes

5. Percentage Decrease

Formula: % Decrease = (Full − Encoded) / Full × 100

Process Steps

  1. Draw/read the grid and note its dimensions (rows × columns)
  2. Compute Full Raster size = total cells
  3. For RLE: scan each row, group consecutive identical values into (value, count) pairs, sum all pairs
  4. For Quadtree: check if entire grid is uniform; if not, split into NW/NE/SW/SE and recurse
  5. For Block: identify largest rectangular uniform blocks and encode as (value, r1, c1, r2, c2)
  6. Calculate % decrease for each method
